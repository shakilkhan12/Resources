Form Validation Library

Using any lightweight framework library you don't need to load the library first you can use the lightweight framework library without loading it

Form Validation Library usage
<?php $this->validation('input_field_name','Label','Rules'); ?>
Validation Rules
  • 1- required
  • 2- not_int
  • 3- int
  • 4- min_len
  • 5- max_len
  • 6- confirm
  • 7- unique
Required rule
<?php $this->validation('input_field_name','Label','required'); ?>
not_int rule

not_int rule means value must be alphabetic character not be integer

<?php $this->validation('input_field_name','Label','not_int'); ?>
int rule

int rule means value must be integer

<?php $this->validation('input_field_name','Label','int'); ?>
min_len rule

after the min_len rule add pipe sign and after immediately add the min length value

<?php $this->validation('input_field_name','Label','min_len|3'); ?>
max_len rule

after the max_len rule add pipe sign and after immediately add the max length value

<?php $this->validation('input_field_name','Label','max_len|8'); ?>
confirm rule

Confirm rule you can use only for password confirm to confirm the two password fields like password field and confirm password field. after the confirm rule add the pipe sign and after immediately add the password field name

<?php $this->validation('input_field_name','Label','confirm|password'); ?>
unique rule

unique rule means to check the email availability in the database table it's available or not in the table, so first add the rule name after the pipe sign and finally add the database table name

<?php $this->validation('input_field_name','Label','unique|users'); ?>
using all rules togather

below is the example how you can use rules togather

<?php $this->validation('input_field_name','Label','required|min_len|4|'); ?>
<?php $this->validation('input_field_name','Label','required|max_len|8|not-int'); ?>
<?php $this->validation('input_field_name','Label','required|confirm|password'); ?>
<?php $this->validation('input_field_name','Label','required|unique|users'); ?>
